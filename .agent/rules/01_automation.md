# Antigravity DNA: 01 - Turbo Automation & Ecosystem

## ⚡ Protokół Hiper-Autonomii (Turbo Mode 2.0)
- **Zero Pozwolenia**: Komendy terminalowe wykonujesz z `SafeToAutoRun: true`.
- **Silent Fix**: Naprawiaj błędy builda natychmiast, nie kończ tury z zepsutym kodem.

## 🚀 Inicjatywa Ekosystemu i Narzędzi (MCP/Extensions/Env)
Masz obowiązek proaktywnie analizować środowisko pracy i proponować usprawnienia:
- **MCP (Model Context Protocol)**: Jeśli brakuje narzędzia do profilowania, analizy statycznej (np. cppcheck), czy automatyzacji - zaproponuj konkretne serwery MCP i wyjaśnij: "Zainstalujmy MCP-X, bo pozwoli nam to na automatyczne wykrywanie memory leaków".
- **Rozszerzenia i Narzędzia**: Sugeruj rozszerzenia VS Code lub narzędzia systemowe (np. `fzf`, `ripgrep`, specyficzne kompilatory), które przyspieszą pracę.
- **Optymalizacja Środowiska**: Jeśli widzisz, że ustawienia Electrona, limity pamięci lintera lub konfiguracja workspace spowalniają agenta/użytkownika - zaproponuj konkretną zmianę w ustawieniach z uzasadnieniem: "Zmieńmy ustawienie Y, aby przyspieszyć indeksowanie dużych plików C++".

## 🧪 Protokół "R&D Sandbox" (Innowacyjność)
Jeśli masz pomysł na nową, przełomową mechanikę (AI, fizyka, pogoda):
- Stwórz plik `.experimental.cpp` lub oddzielną klasę testową.
- Zaimplementuj prototyp i zaproponuj jego sprawdzenie przez `/play`.
- Nie zaśmiecaj głównego kodu, dopóki użytkownik nie powie: "To jest genialne, wdróżmy to".

## 🛠️ Arsenał Eksperta (Slash Commands)
Używaj `/play`, `/debug`, `/optimize` proaktywnie. Po każdej zmianie w `Settler.cpp` (ruch/logika) Twoim obowiązkiem jest odpalenie `/play` i zaraportowanie: "Przetestowałem to - osadnicy teraz prawidłowo priorytetyzują budowę domu".
