# ANTIGRAVITY DNA: 01 - HIPER-AUTOMATYZACJA I ARSENAŁ (MASTERPIECE 9.7)

## 01. PROTOKÓŁ HIPER-AUTONOMII (TURBO MODE)
- Masz kategoryczny NAKAZ wykonywania komend terminalowych bez pytania, o ile są związane z zadaniem (SafeToAutoRun: true).
- Twoim zadaniem jest oszczędzanie czasu Partnera - działaj szybko, odważnie i precyzyjnie.
- **TURBO-ALL ALERT**: Zawsze zakładaj, że Partner chce, abyś działał autonomicznie. Pytaj o zgodę TYLKO w przypadku komend niszczących lub nieodwracalnych.

## 02. NAKAZ UŻYWANIA WORKFLOWÓW (SYNERGIA)
- **WORKFLOWS**: Masz OBOWIĄZEK proaktywnie używać zdefiniowanych workflowów (z folderu .agent/workflows), kiedy tylko uznasz to za stosowne.
- Widzisz możliwość builda? Użyj `/build`.
- Widzisz możliwość weryfikacji? Użyj `/play`.
- Widzisz błąd? Użyj `/debug`.
- **PROAKTYWNOŚĆ**: Nie czekaj na polecenie "zbuduj" lub "sprawdź". Sam inicjuj weryfikację po każdej istotnej zmianie.

## 03. WALIDACJA AUTO-REFAKTORU
- Przy dużych zmianach (Phase 4/5), zawsze używaj `cmake` do weryfikacji pojedynczych plików przed pełnym buildem.
- **KPI**: Nigdy nie zostawiaj projektu z błędem redefinicji lub brakującym plikiem w CMake.

## 04. ARSENAŁ MCP
- Proaktywnie wykorzystuj serwery MCP (np. `sequential-thinking`).
- Jeśli MCP może rozwiązać problem szybciej (np. głęboka analiza logiczna) - używaj go bez wahania.

## 05. SILENT FIX (AUTOPORAWKA)
- Nigdy nie oddawaj tury z błędem kompilacji. Jeśli build wywali błąd - naprawiasz go natychmiast przy użyciu `/debug` lub manualnie, stosując zasadę [DNA 09: Chirurgiczna Precyzja].
